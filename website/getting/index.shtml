How to get Rosegarden

The Rosegarden development team currently only makes Rosegarden available in source code form, to be compiled after download.

Installable binary packages for many major Linux distributions including Mandriva, Debian (package "rosegarden4"), Ubuntu, ALT Linux and PCLinuxOS are available from those distributions' downloadable package repositories. Please consult your distribution's documentation for more details, and check that your distribution provides the latest available version (currently Rosegarden-1.4.0) before downloading. RedHat/Fedora users may wish to consider the excellent PlanetCCRMA audio software package set.

Because of the complexity of Rosegarden and its dependency on numerous other C and C++ libraries and services that vary from one distribution to another, it is sadly not practical at this time for the Rosegarden team to provide installable packages for the many Linux distributions available.

Fervent Software's Studio to Go!

Studio..to go! is a commercial package of music software with a Linux distribution on a single bootable Live CD, with support, produced by Rosegarden developers. Rosegarden is of course included, along with a wide selection of other software all configured and ready to go. This is almost certainly the easiest way to get a complete Rosegarden configuration, particularly if you aren't a Linux user already. Find out more here!

Development code from Subversion

The latest, up-to-the-minute source code in use by Rosegarden developers is always available using the Subversion (svn) source-code control system from SourceForge. Please read the instructions here for more details. Note that this is always a work in progress, that might not build or run.

You can also browse the source code in Subversion here.

Subversion branches available

Several "branches" of the Rosegarden source code are available in Subversion, containing different features for prospective developers or other interested parties. These branches are accessible in the branches/ directory of the svn repository, with the following command :

svn co https://rosegarden.svn.sourceforge.net/svnroot/rosegarden/branches/<BRANCH>/rosegarden rosegarden

replacing <BRANCH> with the name of the branch you want.

The main branch (trunk) is accessible with

svn co https://rosegarden.svn.sourceforge.net/svnroot/rosegarden/trunk/rosegarden rosegarden

Note that the above commands will check out only the Rosegarden source code part of the Subversion repository. If you want to get the whole of the available Rosegarden Subversion tree, including our paltry developer documentation and the HTML for this website, you can remove the "/rosegarden" from the end of the URL part of the above, e.g.:

svn co https://rosegarden.svn.sourceforge.net/svnroot/rosegarden/trunk rosegarden

The current active branches are:

  • trunk - Not a branch strictly speaking, the svn trunk is where most of the development happens. This contains the code that will ultimately become the next official major release of Rosegarden. It is likely to vary between horribly unstable and quite stable, depending on how far away the next major release is.
  • stable_<version> - This is the branch used to collect bug fixes for the next stable minor release. It is branched from the point where the previous major feature release was completed. The most recent stable branch was stable_1_2_4, but use of this branch ended with the release of 1.2.4 and a new stable branch has not yet been opened.
  • glasgow_pitchtracker - This contains extensive custom work to support the University of Glasgow Pitch Tracker project. Note that this branch may be significantly out-of-date compared to the trunk, in all areas other than the pitch tracker support.

Note that we have changed build system (from automake to scons) recently, and some of these branches may still use the old system while others use the new. Check the README for details.


"X11 Rosegarden" 2.1 releases

Before the current Rosegarden project, there was another Rosegarden: a now rather antique program, simpler, less powerful but also less resource-hungry. We're calling it "X11 Rosegarden" here, to distinguish it from the newer Rosegarden. It was originally written for Silicon Graphics IRIX machines, and then ported to Linux and other Unix variants. If you're interested, you can still get it in source code form right here.

  • "X11 Rosegarden" 2.1 patchlevel 4 sources via http (573K).
  • "X11 Rosegarden" 2.1 patchlevel 3 sources via http (570K), and patch from 2.1 patchlevel 2 to patchlevel 3 (7K).
  • "X11 Rosegarden" 2.1 patchlevel 3 for Linux, binaries via http (379K). Built on a Debian 2.2 system.
  • Torsten Duwe's patch to make "X11 Rosegarden" 2.1 use the OSS /dev/sequencer2 API, via http only (24K). This patch also includes some of the fixes in 2.1pl2. It should be applied to the original 2.1 sources; it will not apply correctly to 2.1pl2 or pl3.
  • "X11 Rosegarden" 2.1 for SGI IRIX 6.2, binaries via ftp or via http (501K). This distribution lacks the Petal filter scripting language, because the SGI machine I have access to doesn't have new enough Tcl/TclMIDI components to build it.
  • "X11 Rosegarden" 2.1 for OpenVMS Vax or Alpha, binaries via http in the UK (3.6Mb) -- ported by Patrick Moreau.
  • "X11 Rosegarden" 2.1 sources via ftp or via http (553K).
