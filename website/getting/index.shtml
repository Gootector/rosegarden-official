How to get Rosegarden

The Rosegarden development team currently only makes Rosegarden available in source code form, to be compiled after download.

Installable binary packages for many major Linux distributions including Mandriva, Debian (package "rosegarden4"), Ubuntu, ALT Linux and PCLinuxOS are available from those distributions' downloadable package repositories. Please consult your distribution's documentation for more details, and check that your distribution provides the latest available version (currently Rosegarden-4-1.0) before downloading. RedHat/Fedora users may wish to consider the excellent PlanetCCRMA audio software package set.

Because of the complexity of Rosegarden and its dependency on numerous other C and C++ libraries and services that vary from one distribution to another, it is sadly not practical at this time for the Rosegarden team to provide installable packages for the many Linux distributions available.

Fervent Software's Studio to Go!

Studio..to go! is a commercial package of music software with a Linux distribution on a single bootable Live CD, with support, produced by Rosegarden developers. Rosegarden is of course included, along with a wide selection of other software all configured and ready to go. This is almost certainly the easiest way to get a complete Rosegarden configuration, particularly if you aren't a Linux user already. Find out more here!

CVS development versions

The latest, up-to-the-minute source code in use by Rosegarden developers is always available by anonymous CVS from SourceForge. Please read the instructions here for more details. Note that this is always a work in progress, that might not build or run.

CVS branches

Several "branches" of the Rosegarden source code are available in CVS, containing different features for prospective developers or other interested parties. You can choose one of these by adding the option "-r branchname" to the end of your CVS checkout or update command.

The current active branches are:

  • HEAD - This is the default CVS branch used when no other branch name is given. This contains the code that will ultimately become the next official stable release of Rosegarden, although of course that doesn't mean it'll be stable now. (You can return to this branch after updating from another one, with the "cvs update -A" command.)
  • guitar_chord_editor - This contains experimental development aimed at adding guitar chord fretboard editing and printing within Rosegarden (by Stephen Torri). Note that this branch may be significantly out-of-date compared to HEAD in all areas other than the guitar fretboard code.
  • glasgow_pitchtracker - This contains extensive custom work to support the University of Glasgow Pitch Tracker project. Note that this branch may be significantly out-of-date compared to HEAD in all areas other than the pitch tracker support.
  • experiments - This is a general scratch area for developers, that may contain daring experimental work of all kinds. The only limitation is that it should preferably still compile. Changes to this branch will usually be notified on the Rosegarden development mailing list.

Note that we have changed build system (from automake to scons) recently, and some of these branches still use the old system while others use the new. Check the README for details.


"X11 Rosegarden" 2.1 releases

Before the current Rosegarden project, there was another Rosegarden: a now rather antique program, simpler, less powerful but also less resource-hungry. We're calling it "X11 Rosegarden" here, to distinguish it from the newer Rosegarden. It was originally written for Silicon Graphics IRIX machines, and then ported to Linux and other Unix variants. If you're interested, you can still get it in source code form right here.

  • "X11 Rosegarden" 2.1 patchlevel 4 sources via http (573K).
  • "X11 Rosegarden" 2.1 patchlevel 3 sources via http (570K), and patch from 2.1 patchlevel 2 to patchlevel 3 (7K).
  • "X11 Rosegarden" 2.1 patchlevel 3 for Linux, binaries via http (379K). Built on a Debian 2.2 system.
  • Torsten Duwe's patch to make "X11 Rosegarden" 2.1 use the OSS /dev/sequencer2 API, via http only (24K). This patch also includes some of the fixes in 2.1pl2. It should be applied to the original 2.1 sources; it will not apply correctly to 2.1pl2 or pl3.
  • "X11 Rosegarden" 2.1 for SGI IRIX 6.2, binaries via ftp or via http (501K). This distribution lacks the Petal filter scripting language, because the SGI machine I have access to doesn't have new enough Tcl/TclMIDI components to build it.
  • "X11 Rosegarden" 2.1 for OpenVMS Vax or Alpha, binaries via http in the UK (3.6Mb) -- ported by Patrick Moreau.
  • "X11 Rosegarden" 2.1 sources via ftp or via http (553K).
