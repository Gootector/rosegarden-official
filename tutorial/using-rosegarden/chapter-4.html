4 Editing Notation

Rosegarden stores events in a format that's very notation friendly. Notation is such an integral part of the package that I like to think of the notation as the "real" data, though it is, of course, only one of the ways the data can be presented to you.

One especially unique, powerful and useful feature is the way Rosegarden stores two sets of timings for every event. When you import a MIDI file or record from a keyboard, the raw data gets run through the notation quantizer before being presented to you.

This allows you to maintain a copy of the score for printing, and a copy for playback, affording you the advantages of machine-precise scores while preserving the subtle nuances of human performance.

The feature is new, and not quite complete, however, and there are a few limitations on what you can and cannot do. Look for this to improve for 1.0, but it's rather useful in its present state.

So, let's take a look...

4.1 Opening the Notation Editor

First, load a file into Rosegarden. You might wish to work with link as I have done.

Select at least one segment, then open a notation view by right-clicking on the segment, and choosing "Edit as Notation" from the context submenu.

You may edit multiple segments from any number of tracks simultaneously. Currently the largest caveat here is that if you are editing one or more isolated segments that are separated by a time gap, there is no visual indication in the notation editor to give you an idea where one segment ends, and the other begins. This will, perhaps, be addressed by inserting multi-measure rests in the future, but we've come to no consensus on that issue as yet. In the meantime, I prefer to work with compositions that have long segments where periods of silence have not been cut out.

TIP: You can also configure this to be your default view via Settings -> Configure Rosegarden... -> Default Editor, and thereafter you can get to the notation view by merely double-clicking the desired segment.

4.2 Notation Editor Overview

You'll probably be spending a lot of time here in the notation view. As you'd expect, this is where you go to view and edit your data as standard notation. This editor doesn't quite yet offer all that you might expect from a dedicated notation package, but it's getting there. Most notably absent, there is no facility for flow control. There are no repeats, no codas, no segnos and the like as yet, and I don't honestly expect to see such features until after 1.0.

One of the first things you'll notice is that there are different toolbars here. Before looking, let's synchronize our displays. You may not yet have a Transport Toolbar for this window. Check Settings -> Show Transport Toolbar and the Transport Toolbar should appear. This really should be on by default, if it isn't already, because the notation editor's transport provides some unique functionality that you'd be hard-pressed to do without.

4.2.1 Controlling Playback

The Transport Toolbar that we turned on in the preceding section is the same as in every other window, with two notable exceptions, which is why I've saved it until now. These extra buttons are useful for controlling playback when the playback pointer is outside the segment you're editing, and you want to hear what you're working on.

If you haven't moved it, the playback pointer is at or near the beginning of the composition. This part doesn't come in until much later, so if we hit the Play button now, we can't expect to hear anything for awhile.

The notation editor has its own cursor, which is just like the one that sweeps across the segment canvas in the main window, except that this one is magenta. In order to control playback, you need to be able to manipulate both cursors, and those two extra buttons on the end of the toolbar help you do just that.

Let's move the playback cursor to the beginning of this segment. Click on the ruler above the staff to position the local cursor, just as the ruler in the main window positions the playback

pointer itself. Click the playback pointer to Cursor button to move the playback pointer to the position of the cursor. Now playback will begin from that point.

You can set it back "out of sight" so that playback will begin before the start of the segment

you're editing by shifting it to the left via the button.

4.2.2 Notation Quantization

Those of you familiar with previous versions of Rosegarden will note the absence of the Smoothing slider and associated menu options. As I mentioned at the beginning of this chapter, Rosegarden now maintains two independent values for note duration, making it more easy than ever before to clean up notation, and allowing you to do so without altering the characteristics of the performance.

The Raw Note Ruler shows you the difference between the duration of the note on the staff and the actual timing of the performance data. It may eventually be possible to manipulate the performance duration directly from the ruler, though for the moment it mainly serves to show you how much the written notation differs from the raw timing.
