What is Rosegarden-4?

Rosegarden-4 is the new development series of the Rosegarden MIDI and audio sequencer and musical notation editor. It is free software for Unix and X, running within the KDE desktop environment.

Rosegarden-4 is not a stable release product; it is intended for developers, contributors and other interested parties. If you want a proper version of Rosegarden that works, but that looks old and has a rather limited design, try Rosegarden-2.1.

Current status

The latest developer release of Rosegarden-4 is the rather confusingly-numbered 0.1.4, released on 22 Mar 2002.

It includes complete or partial support for MIDI file I/O, playback and recording, a track editor, score display and editing, and a piano-roll editor. Changes since 0.1.3 include:

  • High (960 ppq) sequencer resolution
  • Properly working Matrix (piano roll) view
  • Much prettier Transport
  • Much improved Looping
  • Undoable Segment actions
  • Undoable and non-destructive Quantization
  • More undoable stuff generally
  • A Metronome
  • Cute coloured MIDI VU-meters with peak hold
  • Segment Parameter Box (Repeat, Quantize, Transpose, Delay)
  • Instrument Parameter Box (Bank select, Program, Channel, Pan, Velocity)
  • Segment (MIDI or Audio containers) splitting and joining
  • The beginnings of a Rosegarden "Studio" concept. This is a MIDI and Audio mapping layer similar to the Logic "Environment". First pass support includes GM patches and some XG extensions.
  • Audio playback (proof of concept bit still buggy)
  • MIDI Recording with count-in and recording segment trail
  • Fixed some leaks in the Sequencer
  • Repeating Segments -- only supported for performance, no visual representation yet
  • Switchable Instrument/Track labels
  • Much faster notation rendering and update
  • Key signature and time signature dialogs in notation view
  • Chord labelling in notation view (due to Randall Farmer)
  • Playback tracking in notation and matrix views
  • More graceful notation rendering from complex performance-type MIDI files
  • Better refresh between concurrent views
  • Switched to a compressed file format

Have a look at some pictures of it in use.

You can download the source package from SourceForge, or obtain more recent source code directly from the CVS repository. In order to build it, you will need a Unix-like operating system, the gcc-2.9x C++ compiler, KDE 2 including the kdemultimedia package, and Qt 2. If you haven't got KDE, see the KDE web site.

The SourceForge site also contains some code documentation and a mailing-list.
