Rosegarden-4 binary downloads

Note: The Rosegarden team currently provides code in source form only. Get the latest release here.

Binary packages are also available, but they are all maintained by other people, so it may take a short while before they are brought up-to-date with each new release of Rosegarden and there's no guarantee of their completeness. Please take a moment to check whether you are really getting the latest release (currently Rosegarden-4-0.9.6) before you download a binary package.

Rosegarden-4 for Mandrake

  • Rosegarden-4 for Mandrake, one of Thac's amazing list of audio and music application RPMs -- click through according to your version of Mandrake and scroll down to find Rosegarden.

Apparently there's also an RPM in the Mandrake Cooker bleeding-edge package repository.

Rosegarden-4 for SuSE

We don't know of any source of up-to-date SuSE packages of Rosegarden-4. SuSE have included Rosegarden-4 in their last two distributions, but neither of the packages worked very well.

We recommend building from source on SuSE: two of the three core Rosegarden developers use SuSE and you should be able to compile Rosegarden out of the box on SuSE 8.2 or 9.0.

Rosegarden-4 for RedHat

  • Rosegarden-4 for RedHat 7.3, 8.0, and 9.0 from Planet CCRMA.

    Planet CCRMA is a well-maintained and easy to use distribution of audio and sound applications for RedHat, which can install everything you need semi-automatically. If you want to set up an audio workstation on RedHat, your best bet is to install the Planet CCRMA system: read the documentation on the Planet site, and go from there.

If you want to run Rosegarden-4 on RedHat but you don't want to use Planet CCRMA, you're probably on your own. It's a difficult distribution to run much modern audio software on, because it doesn't include the ALSA 0.9.x sound drivers.

Rosegarden-4 for Debian

  • Rosegarden-4 is available as part of Debian unstable, thanks to Enrique Robledo Arnuncio.

Rosegarden-4 for your distribution

Several distributions are now including versions of Rosegarden, if not in the base distribution then in an extended package repository. For example, Mandrake, SuSE, Debian and the Russian language ALT Linux all make Rosegarden packages available to their users.

If we're not packaged as part of your favourite distribution then please get in touch with your distribution manufacturer and ask why not! If you've made an RPM or package for any distribution then please let us know so that we can link it here.

Rosegarden-4 source code

If you can't find a package to suit your system them you can still build Rosegarden-4 from source code. The latest release is shown below:

  • Rosegarden-4 release 0.9.6 sources, 12th January 2004, for download from the SourceForge page.
  • Rosegarden-4 CVS from the SourceForge repository; this is the latest bleeding-edge, work-in-progress and might not build or run but will have all the latest features as discussed on the rosegarden-devel mailing list.

The SourceForge site also contains some code documentation and a mailing-list.

See the requirements page for details on how to build the source code.

Design reports

Rosegarden started life as a university project. These are the original reports Chris and Andy produced for that project, way back in the first half of 1994. They probably still constitute the best description of the internals of the Rosegarden 2.1 application, although many details are now out-of-date.

  • Notation editor design document (212K, gzipped PostScript).
  • Sequencer design document (691K, Acrobat PDF).
  • Interlock tool-communication framework design document (96K, Acrobat PDF).
