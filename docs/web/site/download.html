Rosegarden-4 releases

We currently only make distributions available in source code form.

  • Rosegarden-4 release 0.8, 4 October 2002: get the sources from the SourceForge page.

Rosegarden-4 is also included in the Debian unstable ("sid") distribution, with the package name rosegarden4 (thanks to Enrique Robledo Arnuncio).

If you'd like to support Rosegarden-4 but you aren't a developer, another way to contribute is with a financial donation.
A professional sequencer costs hundreds of pounds. We haven't written one yet, but we aren't asking for hundreds of pounds either. Twenty quid or so would be great.
Thanks!

If you've created another binary package, please get in touch with one of the authors so we can link it up here.

You can also obtain the most recent source code (which might not work) directly from the CVS repository.

In order to build Rosegarden-4, you will need a Unix-like operating system, the gcc-2.9x or 3.x C++ compiler, KDE 2.2 with Qt2 or KDE3 wth Qt3. See the FAQ for solutions to a few build and installation problems.

Other related software Rosegarden relies on (not essential - you can build without it) :

  • Alsa 0.9 or better (highly recommended - much better sound rendering than with KDE's aRts), but your distribution most likely carries it already, except RedHat which as of 8.0 still ships Alsa 0.5)
  • JACK, (Jack Audio Connection Kit) for audio.
  • LADSPA, (Linux Audio Developer's Simple Plugin API), for audio plugins.

The SourceForge site also contains some code documentation and a mailing-list.

Design reports

Rosegarden started life as a university project. These are the original reports Chris and Andy produced for that project, way back in the first half of 1994. They probably still constitute the best description of the internals of the application, although many details are now out-of-date.

  • Notation editor design document (212K, gzipped PostScript).
  • Sequencer design document (691K, Acrobat PDF).
