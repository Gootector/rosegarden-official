Hardware requirements

Rosegarden-4 is a MIDI and audio sequencer and notation editor and its system requirements differ according to what task you're using it for. Editing, playing and recording MIDI is a less resource hungry mode of operation than recording or playing audio. The use of multiple audio plugins or soft-synths will especially impact system performance, and it's worth bearing your projected usage in mind when specifying your machine.

For a PC, minimum hardware requirements for running Rosegarden-4 for MIDI use with simple editing and playback/recording is a 400MHz system with 64MB of memory. A recommended system is an 800MHz system with 256MB of memory - with this setup you should have enough power to sequence MIDI and audio with a small number of audio plugins. For serious audio use and/or use with a serious number of high-quality soft synths, you'll need significantly more processor power.

Software requirements

Rosegarden-4 runs on Linux, so you'll certainly need that. It uses the KDE 3 libraries, so you will need KDE 3.x with Qt 3. You don't actually need to use the KDE desktop, you just need the library packages installed on your system.

If you want to hear sound from your system then you will also need to configure a soundcard supported by the ALSA project.

Audio support is provided by the JACK sound server.

If you're using an external MIDI device (sound module or keyboard) then you'll also need the appropriate leads and cables. For help setting up your home studio for easy audio and MIDI recording then you can consult Bownie's DIY Guide to Recording.

Please also consult the FAQ for more information.

Building from source code

In order to build Rosegarden-4, you will need a Unix-like operating system, the gcc-2.9x or 3.x C++ compiler, KDE 3.x with Qt3. See the FAQ for solutions to a few build and installation problems.

Other related software that you may wish to build with includes:

  • ALSA 0.9 or better (highly recommended - much better sound rendering than with KDE's aRts), but your distribution most likely carries it already, except RedHat which as of 8.0 still ships Alsa 0.5)
  • JACK (Jack Audio Connection Kit) for audio.
  • LADSPA (Linux Audio Developer's Simple Plugin API), for audio plugins.
